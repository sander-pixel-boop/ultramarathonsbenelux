fix: Use correct races.json output path in scraper

The scraper was writing its output to `races.json` instead of `data/races.json`. This caused the main app (which reads from `data/races.json`) to use an older, truncated version containing only 10 races. This commit restores `data/races.json` from the full `races.json` output, updates `scripts/scraper.js` to correctly use `data/races.json`, and removes the incorrect root `races.json` file.
